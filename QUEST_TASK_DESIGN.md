# 🗡️ クエスト型タスク 仕様設計書 v1.0
## Saikou! Quest System Design

---

## 📊 現状分析（現行システム）

| 項目 | 現状 |
|------|------|
| タスク構造 | task_text（テキスト）+ genre_id + difficulty のみ |
| 完了方法 | チェックボックスON/OFFだけ |
| XP計算 | 1タスク=3XP固定、全完了ボーナス+3 |
| ストリークボーナス | streak≥3で+1 〜 streak≥100で+12 |
| 1日最大XP | 3タスク全完了 + streak100日 = 3×3+3+12 = **24XP** |
| レベル1→2 | 約100XP（= 約5日で到達） ←「簡単すぎる」原因 |

### 問題点
- チェックだけで完了できる → 嘘チェックし放題
- 目標値・達成根拠がない → 努力の可視化ができない
- 投稿・シェア機能がない → ソーシャル要素ゼロ
- XPが軽すぎる → レベルアップが安易

## 🎮 クエスト型タスク 全体設計

### タスクタイプ定義

| タイプ | コード | 特徴 | 完了方法 |
|--------|--------|------|---------|
| 回数型 | `COUNT` | 腕立て30回、英単語50個 | 達成数を入力して完了 |
| 時間型 | `TIME` | 英語30分、読書20分 | STARTボタン→タイマー→完了 |
| 実行型 | `EXEC` | 部屋を掃除、母に電話 | 完了ボタンを押すだけ |

## ⚡ XP設計（改訂版）── 「簡単すぎない」バランス

### 現行の問題
- 1タスク3XP × 3タスク = **9XP/日**（最大24XP）
- Lv1→2が約100XP ≒ **4〜5日で到達**
- ストリーク継続しなくてもすぐレベルアップ

### 改訂XP設計

#### タスク基本XP（タイプ×難易度）

| タイプ | 目標達成 | 1.5倍達成（目標×1.5以上） | 2倍達成（目標×2以上） |
|--------|---------|----------------------|------------------|
| 実行型 | 8 XP | — | — |
| 回数型 | 10 XP | 14 XP | 18 XP |
| 時間型 | 12 XP | 17 XP | 22 XP |
| 未達成 | 0 XP | — | — |

#### ボーナスXP

| 条件 | ボーナス |
|------|---------|
| 写真付き投稿 | ×1.5倍（小数切り捨て） |
| 全タスク完了（3タスク） | +10 XP |
| ストリーク 7日 | +5 XP/日 |
| ストリーク 30日 | +12 XP/日 |
| ストリーク 100日 | +25 XP/日 |

#### 1日の最大XP目安（3タスク・ストリーク100日・全写真投稿）
```
時間型×3 = 22×1.5×3 = 99XP
全完了ボーナス = 10XP
ストリーク = 25XP
合計 ≈ 134XP（最高効率）

通常（実行型×3、写真なし）= 8×3 + 10 = 34XP
```

#### レベルアップ必要XP（現行→改訂案）

| Lv | 現行必要XP | 改訂必要XP | 通常プレイでの日数目安 |
|----|-----------|-----------|------------------|
| 1→2 | 100 | **200** | 約7日 |
| 5→6 | 500 | **800** | 累計約1ヶ月 |
| 10→11 | 1000 | **2000** | 累計約3ヶ月 |
| 20→21 | 2000 | **5000** | 累計約6ヶ月 |
| 50→51 | 5000 | **15000** | 累計約2年 |

**→ 真剣に毎日やって数ヶ月でLv10、1年以上でLv20〜30という設計**

## 🗄️ DB追加カラム（tasksテーブル）

```sql
ALTER TABLE tasks ADD COLUMN task_type TEXT DEFAULT 'EXEC';
-- 'COUNT' | 'TIME' | 'EXEC'

ALTER TABLE tasks ADD COLUMN target_value INTEGER DEFAULT NULL;
-- 回数型: 目標回数、時間型: 目標分数、実行型: NULL

ALTER TABLE tasks ADD COLUMN unit TEXT DEFAULT NULL;
-- '回' | '分' | NULL
```

### 新規テーブル: task_logs（タスク実行記録）

```sql
CREATE TABLE task_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  task_id UUID REFERENCES tasks(id),
  date DATE NOT NULL,
  task_type TEXT NOT NULL,
  target_value INTEGER,
  actual_value INTEGER,   -- 実際の達成値（回数 or 分）
  xp_earned INTEGER DEFAULT 0,
  completed_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 新規テーブル: posts（タイムライン投稿）

```sql
CREATE TABLE posts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  task_log_id UUID REFERENCES task_logs(id),
  comment TEXT,
  photo_url TEXT,          -- Supabase Storage URL
  xp_bonus_applied BOOLEAN DEFAULT FALSE,
  likes_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## ⚠️ 重要な設計原則

> **「努力した人が報われる」設計にする**
> 
> - 写真なし実行型は最低XP → サボり抑止
> - 時間型はSTART必須 → 嘘チェック防止
> - レベルアップを重くする → 達成感の強化
> - 他人の努力が見える → 継続モチベーション
