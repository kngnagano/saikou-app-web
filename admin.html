⚙️ Supabase Realtime 有効化ガイド

通知がリアルタイムで届くには、Supabase の Publications 設定が必要です。
以下の手順を順番に実施してください。

1

Supabase Dashboard を開く

https://supabase.com/dashboard を開き、
プロジェクト mthfqqqukuvueprdokiq を選択

2

Database → Publications を開く

左メニューの Database → Publications をクリック
⚠️ 「Replication」ではなく「Publications」です！

📍 メニューの場所：
左サイドバー → Table Editor の下 → Database セクション → Publications

3

supabase_realtime をクリック

一覧に表示される supabase_realtime をクリックして詳細を開く

4

notifications テーブルを ON にする

Source tables セクションで notifications テーブルのトグルを ON にする
（daily_status も ON にすると フレンド達成通知も動く）

✅ これで admin.html から通知を送ると、アプリにリアルタイムで届くようになります！

🚨 現在の問題: users テーブルが読めない

ログを見ると 「ユーザー 0 件取得」 が発生しています。
これは users テーブルの RLS が anon キーの SELECT を拒否しているためです。

▼ Supabase SQL Editor でこれを実行してください：

-- ① users テーブル: anon が SELECT できるようにする
CREATE POLICY "admin_can_read_users"
ON public.users FOR SELECT
USING (true);

-- ② daily_status テーブル: anon が SELECT できるようにする
CREATE POLICY "admin_can_read_daily_status"
ON public.daily_status FOR SELECT
USING (true);

-- ③ notifications テーブル: anon が INSERT できるようにする
CREATE POLICY "admin_can_insert_notifications"
ON public.notifications FOR INSERT
WITH CHECK (true);

-- ④ notifications テーブル: anon が SELECT できるようにする
CREATE POLICY "admin_can_read_notifications"
ON public.notifications FOR SELECT
USING (true);

-- ⑤ notifications テーブル: anon が UPDATE できるようにする（既読化）
CREATE POLICY "admin_can_update_notifications"
ON public.notifications FOR UPDATE
USING (true);
✅ コピーしました！Supabase SQL Editor に貼り付けて実行してください

⚠️ すでに同名ポリシーがある場合はエラーになります。
その場合は DROP POLICY IF EXISTS で削除してから再実行するか、
Supabase Dashboard の Authentication → Policies から既存ポリシーを確認してください。

🚀 Edge Function デプロイ（アプリを閉じていても届く）

現在のステータス: 確認中...

A. Supabase CLI をインストール

npm install -g supabase

B. VAPID キーを生成・登録

# Node.js で VAPID キーを生成
npx web-push generate-vapid-keys
# 生成されたキーを Supabase に登録
supabase secrets set VAPID_PUBLIC_KEY=<公開鍵>
supabase secrets set VAPID_PRIVATE_KEY=<秘密鍵>
supabase secrets set VAPID_SUBJECT=mailto:your@email.com

⚠️ 公開鍵は index.html の VAPID_PUBLIC_KEY にも設定してください

C. Edge Function をデプロイ

# プロジェクトにログイン
supabase login
supabase link --project-ref mthfqqqukuvueprdokiq
# デプロイ
supabase functions deploy send-push-notification

D. push_subscriptions テーブルを作成

CREATE TABLE IF NOT EXISTS public.push_subscriptions (
  id         uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id    text NOT NULL UNIQUE,
  endpoint   text NOT NULL,
  p256dh     text NOT NULL,
  auth       text NOT NULL,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);
ALTER TABLE public.push_subscriptions ENABLE ROW LEVEL SECURITY;
CREATE POLICY "service_role_all" ON public.push_subscriptions
  USING (true) WITH CHECK (true);
