# メール＋パスワード認証実装方針

## 変更点

### 1. データベース構造
- `users` テーブルに `email` カラムと `auth_user_id` カラムを追加
- Supabase Auth のユーザーID と紐付け

### 2. 認証フロー

#### サインアップ
1. メールアドレス＋パスワードで Supabase Auth にユーザー登録
2. 認証成功後、`users` テーブルにレコード作成
3. 招待コード（任意）があれば検証
4. 自動生成された招待コードを付与

#### ログイン
1. メールアドレス＋パスワードで Supabase Auth 認証
2. 認証成功後、Auth のユーザーID から `users` テーブルのレコード取得
3. アプリの状態を読み込み

#### パスワードリセット
1. メールアドレス入力
2. Supabase Auth がリセットメール送信
3. リセットリンクから新しいパスワード設定

## 実装ステップ

1. ✅ ログイン・サインアップ画面の UI 更新
2. ⏳ `handleSetup` 関数を書き換え（Supabase Auth 使用）
3. ⏳ `handleLogin` 関数を書き換え（Supabase Auth 使用）
4. ⏳ パスワードリセット機能追加
5. ⏳ Supabase でメール認証設定

## 注意事項

- 既存ユーザーの移行は手動で行う必要あり
- 招待コードシステムは維持（オプション）
- LocalStorage にセッション情報を保存
