# フレンド承認システム テストガイド

## 🧪 テストシナリオ

### 準備
1. 2つのアカウントを用意（アカウントA、アカウントB）
2. それぞれログイン可能な状態にする

---

## シナリオ1: 基本的なフレンド承認フロー

### ステップ1: アカウントAからBへリクエスト送信
1. アカウントAでログイン
2. 「招待」タブを開く
3. アカウントBの招待コードを入力
4. 「追加」ボタンをクリック
5. 「〇〇さんを追加しました！」アラート表示を確認
6. 「フレンド」タブを開く
7. **確認ポイント**:
   - ⏳ 承認待ち セクションにアカウントBが表示される
   - 「相手の承認待ちです」テキストが表示される
   - カードが半透明で表示される

### ステップ2: アカウントBで承認
1. アカウントBでログイン
2. 「フレンド」タブを開く
3. **確認ポイント**:
   - 📩 承認待ちのリクエスト セクションが表示される
   - アカウントAのリクエストが表示される
   - 「承認」「拒否」ボタンが表示される
4. 「承認」ボタンをクリック
5. 「フレンドリクエストを承認しました！」アラート確認
6. **確認ポイント**:
   - 📩 承認待ちのリクエスト セクションが消える
   - フレンド セクションにアカウントAが表示される
   - アカウントAの継続日数が表示される

### ステップ3: アカウントAで確認
1. アカウントAの画面をリロードまたはフレンドタブを再度開く
2. **確認ポイント**:
   - ⏳ 承認待ち セクションが消える
   - フレンド セクションにアカウントBが表示される
   - お互いにフレンドリストに表示される

---

## シナリオ2: フレンドリクエスト拒否

### ステップ1: アカウントCからAへリクエスト送信
1. 新しいアカウントC作成
2. アカウントAの招待コードでリクエスト送信

### ステップ2: アカウントAで拒否
1. アカウントAでログイン
2. 「フレンド」タブを開く
3. アカウントCのリクエストを確認
4. 「拒否」ボタンをクリック
5. 確認ダイアログで「OK」
6. 「フレンドリクエストを拒否しました」アラート確認
7. **確認ポイント**:
   - アカウントCのリクエストが消える
   - フレンドリストには追加されていない

### ステップ3: アカウントCで確認
1. アカウントCの画面をリロード
2. **確認ポイント**:
   - ⏳ 承認待ち セクションが消える
   - フレンドリストにアカウントAは表示されない

---

## シナリオ3: 複数リクエストの管理

1. アカウントAで3人のユーザー（D, E, F）からリクエストを受信
2. **確認ポイント**:
   - 📩 承認待ちのリクエスト (3) と表示される
   - 3つのリクエストが縦に並んで表示される
3. Dを承認、Eを拒否、Fはそのまま
4. **確認ポイント**:
   - フレンドリストにDが表示される
   - 📩 承認待ちのリクエスト (1) と表示される
   - Fだけが承認待ちに残る

---

## エッジケース

### ケース1: 同じユーザーに複数回リクエスト
1. アカウントAからBへリクエスト送信
2. 承認前に再度同じ招待コードを入力
3. **期待動作**: 「すでにフレンドです」または類似のエラー表示

### ケース2: 既存フレンドの削除後の再追加
1. アカウントAとBが既にフレンド
2. アカウントAがBを削除
3. 再度アカウントAからBへリクエスト送信
4. **期待動作**: 再度承認プロセスが発生

### ケース3: リクエスト承認中のエラーハンドリング
1. ネットワークを切断してリクエスト承認を試行
2. **期待動作**: エラーアラート表示、データは変更されない

---

## データベース確認

Supabase ダッシュボードで以下を確認：

### friends テーブル
```sql
SELECT * FROM friends WHERE user_id = '<アカウントAのID>' OR friend_user_id = '<アカウントAのID>';
```

**確認ポイント**:
- 承認待ちリクエスト: `status = 'pending'`
- 承認済みフレンド: `status = 'approved'`
- 双方向関係: A→B と B→A の両方のレコードが存在

## トラブルシューティング

### 問題: リクエストが表示されない
- ブラウザコンソールでエラー確認
- RLS設定確認（SELECT権限）
- キャッシュクリア後再ロード

### 問題: 承認ボタンが動作しない
- ブラウザコンソールでJavaScriptエラー確認
- `window.approveFriend` が定義されているか確認
- RLS設定確認（UPDATE権限）

### 問題: 双方向関係が作成されない
- `approveFriend` 関数のログ確認
- INSERT権限のRLS確認
- エラーアラートの内容確認
